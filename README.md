# MBA-HCM Assistant

This repository is for organizing Hannah Oncada’s MBA Healthcare Management coursework, academic projects, presentation templates, previous coursework, and final submission files.

This repository is not limited to one semester. It will include current coursework, future coursework, and selected previous-semester projects that are related to healthcare management, business, leadership, operations, service quality, finance, process improvement, and professional growth.

The repository should stay simple, useful, and aligned with professor instructions, course syllabi, rubrics, Canvas announcements, and assignment prompts.

## Main Rule

Each course must be aligned with its syllabus.

Canvas and professor announcements are the final guide for updated deadlines, assignment changes, rubrics, and submission instructions.

Do not overbuild the repository. Create folders and files only when they help with a real assignment, rubric, project, presentation, or professor instruction.

## Purpose

The purpose of this repository is to keep MBA Healthcare Management schoolwork organized in one place.

It will help organize:

* Case studies
* One-minute memos
* Discussion posts
* Journals
* Six Sigma project sections
* Startup project work
* Accounting and finance assignments
* Final papers
* PowerPoint presentations
* AI use notes
* Previous coursework
* Final submission copies

## Writing Style

All schoolwork should be written in a style that is:

* Clear
* Simple
* Professional
* Graduate-level
* Easy to understand
* Human and not robotic
* Connected to healthcare management, business, leadership, service, operations, or financial responsibility when appropriate

Personal experience may be included when it supports the assignment, but the writing should not sound only from a nursing point of view.

## Presentation Master Format

All PowerPoint presentations should use the same personalized master format, no matter what course the presentation is for.

The master PowerPoint file should be saved here:

```text
templates/presentation-master/PowerPoint Master.pptx
```

This file will be used as the base format for all future presentations.

The purpose of the master format is to keep all presentations:

* Consistent
* Professional
* Organized
* Easy to read
* Aligned with Hannah Oncada’s academic style

When creating a new presentation, start from the master PowerPoint file first. Then adjust the number of slides based on the professor’s instructions, rubric, assignment prompt, and required presentation length.

Do not create extra slides just to make the presentation longer. The number of slides should depend on the assignment requirement.

## Presentation Workflow

Use this process for any PowerPoint project:

```text
1. Read the professor’s presentation instructions and rubric.
2. Open the PowerPoint Master file.
3. Create only the slides required for the assignment.
4. Keep the layout clean and consistent.
5. Use short slide titles and simple wording.
6. Add speaker notes only if required.
7. Check that the presentation follows the course rubric.
8. Save the final file using the correct file naming rule.
```

Final PowerPoint files should follow this format:

```text
HannahOncada [Presentation Title].pptx
```

Example:

```text
HannahOncada Six Sigma Project Presentation.pptx
```

If a presentation belongs to a specific course, save it in that course folder.

Example:

```text
current-courses/capstone/presentations/
```

If the course does not require a presentation, do not create a presentation folder for that course.

## How to Process Each Assignment

Use this process for every school project.

### Step 1: Confirm the Assignment

Before starting, collect the basic information:

```text
Course:
Professor:
Assignment title:
Assignment type:
Due date:
Required length:
Required format:
Submission method:
Required sources:
Rubric provided? Yes/No
AI use allowed or required? Yes/No
```

### Step 2: Follow the Professor First

Before writing, check:

* Syllabus
* Canvas announcement
* Assignment prompt
* Rubric
* Lecture notes or PowerPoint
* Required article or textbook
* Professor’s special instructions

Do not add unnecessary sections if the professor did not ask for them.

### Step 3: Create the Work in the Correct Folder

Use the correct course folder.

Example for Capstone case study:

```text
current-courses/capstone/case-studies/class-4-speed-reading/
```

Example for Six Sigma project section:

```text
current-courses/six-sigma/project-sections/project-1-charter/
```

Example for Capstone presentation:

```text
current-courses/capstone/presentations/startup-project-presentation/
```

### Step 4: Draft the Assignment

Most written assignments should follow this simple structure:

```text
1. Main question or topic
2. Simple explanation of the course concept
3. Application to the case, problem, or assignment
4. Connection to healthcare management, business, leadership, operations, service, or finance when appropriate
5. Personal experience only if it supports the assignment
6. Conclusion or recommendation
7. References if required
```

### Step 5: Review Before Finalizing

Before submission, check:

```text
Did I answer the assignment question?
Did I follow the rubric?
Did I follow the syllabus and Canvas instruction?
Did I keep the writing clear and simple?
Did I avoid unnecessary jargon?
Did I connect the topic to business or healthcare management when appropriate?
Did I avoid confidential patient, resident, facility, or employer information?
Did I cite sources if required?
Did I include AI use notes if required?
```

### Step 6: Save the Final Version

Final school files should be saved in the correct course folder and copied to the course final-submissions folder.

Examples:

```text
current-courses/capstone/final-submissions/
current-courses/six-sigma/final-submissions/
current-courses/accounting/final-submissions/
```
